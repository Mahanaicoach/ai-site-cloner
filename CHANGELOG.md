# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.2.0] - 2026-07-21

### Added

- Multi-tool portability: instruction files and skills for 12 agent tools (Claude Code, Cursor, Windsurf, GitHub Copilot, Codex CLI, Gemini CLI, Cline/Roo Code, Continue, Amazon Q, Augment Code, OpenCode, Aider), regenerated by a sync script
- Docker packaging: multi-stage production `Dockerfile` (Next.js standalone on `node:22-alpine`), Playwright-based `Dockerfile.dev` with Chromium preinstalled, `docker-compose.yml` with prod + dev services, and `.dockerignore`
- Auto-installing Chromium — extraction scripts install the browser on first run, no manual `npx playwright install` step
- Bug report and feature request issue templates, pull request template, and this changelog
- Repository metadata in `package.json` (repository, homepage, bugs)

### Changed

- Performance overhaul of the extraction pipeline:
  - One-shot page recon (`page.mjs`) — full extraction in 3 page loads per page instead of one load per collector
  - Replaced blind `networkidle` waits with a hard cap plus precise settle signals (fonts, images, animation frames) running concurrently
  - Disk-backed response cache shared across scripts, so repeat visits never re-fetch the original site
  - 8-way concurrent asset downloads
  - QA compares all viewports of a page in one call, cropping section shots out of a single full-page capture and caching original-side screenshots
- Precision improvements to extraction and QA:
  - Animations frozen at their end state before screenshots, so diffs measure layout instead of motion
  - Pixel-diff mismatches localized to 10 horizontal bands per section with absolute page coordinates, so builder agents get told exactly where to look
  - Single-pass batched style walker that is pseudo-element-aware (`::before`/`::after` styles captured alongside the owning element)
  - `<canvas>` content captured via `captureStream` on the shared browser instead of being left as a hole

## [0.1.0] - 2026-07-21

### Added

- Initial template: clone any multi-page website into a clean Next.js 16 + Tailwind CSS v4 + shadcn/ui codebase
- Playwright extraction scripts — crawl, computed styles, real CSS rules, design tokens, responsive breakpoints, assets, screenshots (`scripts/extract/`)
- Spec-driven parallel builds: per-component spec files linted by `lint-spec.mjs`, built by parallel agents, tracked in `docs/research/manifest.json`
- Scored visual QA: pixel-diff comparison against the original site (`scripts/diff.mjs`)
- `/clone-website` and `/restyle` skills driving the full pipeline
- Uploadthing example clone in `examples/`
